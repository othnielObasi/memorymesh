# Branding

## Product name

MemoryMesh

## Why this name

MemoryMesh is broader than a coding-agent name. It describes a connected work-memory layer for many agent types: coding, research, support, operations, compliance, and enterprise automations.

The name also fits Cognee’s hybrid graph-vector memory angle: memories are not isolated notes; they are connected task contracts, decisions, evidence, failures, checkpoints, and improvement lessons.

## Tagline

Never-forget work memory for long-running AI agents.

## One-line pitch

MemoryMesh gives agents durable memory of work, so they can recover, continue, and improve after context loss.

## What the coding demo represents

The coding agent is the proof case, not the whole product. It proves the broader claim with objective evidence: tests fail before recovery, Cognee memory restores context, the agent patches code, and tests pass after recovery.
